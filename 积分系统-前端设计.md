# 积分系统 · 前端设计文档

## 1. 概述

四人对战积分系统。每局同时记录**奖金分配**和**胜负结果**。奖金为每人独立计数，可为零或多个，不设上限；胜负固定为 2 胜 2 负。

---

## 2. 核心规则

### 2.1 奖金（Bonus）

- 每位玩家有独立奖金计数器，初始为 0
- 通过 `[+1]` / `[-1]` 按钮增减计数，**不限次数**
- 每个奖金独立结算：
  - 获得者：**+3**
  - 其他三人：各 **-1**

**单人奖金分公式：**

```
奖金分 = 本人奖金数 × 3 − 其他三人奖金总和 × 1
```

> 等价理解：每个奖金给获得者 +3，给其他三人各 -1，全部累加即得。

**校验：** 奖金分总和恒为零（零和机制）

**示例：A × 3，B × 1，C × 0，D × 0**

| 计算项 | A | B | C | D |
|-------|:---:|:---:|:---:|:---:|
| 奖金数 | 3 | 1 | 0 | 0 |
| 本人收入 | +9 | +3 | 0 | 0 |
| 扣他人 | −1 | −3 | −4 | −4 |
| **奖金分** | **+8** | **0** | **−4** | **−4** |
| | 🟢 总和 = **0** ✅ |

**更多示例：**

| A | B | C | D | 计算结果 |
|:-:|:-:|:-:|:-:|---------|
| 1 | 0 | 0 | 0 | **+3** / -1 / -1 / -1 |
| 1 | 1 | 0 | 0 | **+2** / **+2** / -2 / -2 |
| 2 | 1 | 0 | 0 | **+5** / **+1** / -3 / -3 |
| 3 | 3 | 3 | 3 | 0 / 0 / 0 / 0 |
| 5 | 0 | 0 | 0 | **+15** / -5 / -5 / -5 |

### 2.2 胜负结果

- 格局固定：**2 人胜、2 人负**
- 分值有两种互斥类型：

| 类型 | 胜方每人 | 负方每人 |
|------|---------|---------|
| **大胜** | **+2** | **-2** |
| **小胜**（即"两人赢两人输"）| **+1** | **-1** |

### 2.3 本局合计

```
本局得分 = 奖金分 + 胜负分
```

## 3. 总计面板

累计面板展示跨局汇总数据。每位玩家按**四项**展示：

```
+9  |  -3  |  +2  |  = 8
奖金收入 |被他人扣分|胜负得分|总分
```

| 明细项 | 含义 | 累计公式 |
|--------|------|---------|
| **奖金收入** | 自己的奖金累计所得 | 累计奖金数 × 3 |
| **被他人扣分** | 被他人奖金累计扣分 | 累计他人奖金数 × 1 |
| **胜负得分** | 胜局累计得分 | 大胜胜局×2 + 小胜胜局×1 |
| **总分** | 前三项代数和 | 收入 − 扣分 + 得分 |

**色彩规则：** 正值 🟢 绿色 / 负值 🔴 红色 / 零值 ⚪ 灰色

### 交互流程

1. **选择胜负类型**：顶部大胜或小胜（单选）
2. **设置奖金**：每人通过 `[−] [+]` 调整奖金数量，**无上限**
   - 每次点击 `[+]` → 该玩家奖金数 +1，自动更新四人分差
   - 每次点击 `[−]` → 该玩家奖金数 −1（不低于 0）
3. **标记胜方**：点击两人设为 `[胜]`，其余两人自动 `[负]`
4. **实时计算**：所有分数实时更新，正/负值自动着色
5. **确认** → 累计入总计面板，进入下一局

## 5. 数据模型

```typescript
type OutcomeMode = 'big_win' | 'small_win'

interface PlayerRoundData {
  id: string
  /** 本局奖金数量（≥ 0，无上限） */
  bonusCount: number
  /** 本局胜负 */
  isWinner: boolean
}

interface GameRound {
  players: PlayerRoundData[]
  outcomeMode: OutcomeMode
}

/** 本局计算 */
interface RoundBreakdown {
  bonusSelf: number           // bonusCount × 3
  bonusFromOthers: number     // -(其他三人 bonusCount 之和)
  outcomeScore: number        // 满足胜负条件后的 ±2 或 ±1
  roundTotal: number          // 三者之和
}

/** 累计 */
interface CumulativeScore {
  bonusEarned: number         // 累计奖金收入（∑bonusCount × 3）
  bonusLost: number           // 累计被他人扣分（∑他人奖金）
  outcomeNet: number          // 累计胜负净得分
  grandTotal: number          // 三项代数和
}
```

## 6. 约束校验（前端）

| 规则 | 校验 |
|------|------|
| 奖金数量 ≥ 0 | 每人的 `bonusCount ≥ 0` |
| 奖金分总和归零 | `∑bonusSelf + ∑bonusFromOthers === 0` |
| 胜负类型二选一 | `outcomeMode` 有值 |
| 胜者恰好两人 | `isWinner` true 计数 === 2 |
| 负者恰好两人 | `isWinner` false 计数 === 2 |
| 全员可确认 | 胜负类型已选，胜者已标记 |
